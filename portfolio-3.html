01 · Infrastructure

AI 마케팅 운영체제

병목
GA도 깨져 있었고, 운영체제도 없었다.
가설은 세웠지만 검증할 도구가 없었다.
가설
AI를 워크플로우로 묶으면, 1인이 팀 규모로 일한다.
어떻게
  • 측정 인프라 (GA4 + CAPI + PostHog + 채널별 유입 시트)도 함께 깔았다 → 축 2·3 진단의 토대
  • 사례 1. 정·반·합 콘텐츠 파이프라인
  • 사례 2. 사내 지식 Ontology화
결과
월 2회 발행
→
주 3회 (AI 2회 + 전환용 1회)
자료 매번 검색
→
ontology에서 즉시 호출
리드 실험 시도 못 함

리드 확보용 광고·CRO 실험 가능

미니 사례 1. 정·반·합 콘텐츠 파이프라인

가설
AI 간 정·반·합 시스템을 구축하면, 콘텐츠의 양과 질을 동시에 잡을 수 있다.
어떻게
  1. 수기 워크플로우 적용
    정 (正)
    Claude — 초안 작성. 구조·논리·SEO. 긴 호흡과 일관성 강점
    반 (反)
    Gemini — 비판 검토. 한국어 톤·자연스러움·디테일 교정
    합 (合)
    본인 — 통합 편집. 충돌 시 사람이 결정 (구조: Claude 우선 / 톤: Gemini 우선 / 최종: 본인)
  2. 양과 질 동시 유지 검증 — 월 2회 → 주 3회 페이스 + 콘텐츠 퀄리티 유지 확인
  3. 검증된 워크플로우를 시스템으로 굳힘 — Claude API → Gemini API → 통합 단계 자동화
결과
  • 월 2회 → 주 3회 발행 (AI 2회 + 전환용 1회)
  • 양과 질 동시 유지 (질 검증은 축 2에서)
  • 검증된 워크플로우 → 자동 시스템으로 안정 운영
영감 출처 — Oh My OpenAgent의 다중 모델 패턴.

미니 사례 2. 사내 지식 Ontology화

가설
사내 지식을 ontology로 외재화하면, 사내 데이터가 통일되어
일관성 있고 효율적인 운영이 가능해진다.
어떻게
  1. Slack·Notion 353건 + Founder/팀원 인터뷰 등 raw intelligence 전수 수집
  2. /brainstorming 커맨드로 도메인별 구조 도출
  3. Ontology 11개 도메인 정의 — 회사·팀·제품·기술·시장·트랙션·투자·로드맵·고객·브랜드·경쟁사
  4. Analysis 7개 작성 — UE·해자·성장진단·SaaS전환·GTM·글로벌·투자준비
  5. 17 에이전트 + 19 플러그인 + 13 마케팅 스킬에 ontology 연결
결과
  • 사내 데이터 353건 → ontology 11 + analysis 7로 통일
  • /route, /ask 한 줄로 회사 맥락 즉시 호출, 자료 검색 시간 단축
  • AI 도구가 회사 맥락 일관 답변 생산
  • 가설 검증 = 데이터 통일 → 일관성·효율 운영 확보
02 · Demand

콘텐츠 — 전략·깊이

병목
콘텐츠 양산만으로는 SEO도 전환도 신뢰도 안 만들어졌다.
가설
양산이 아니라 코어 콘텐츠 (BOFU, Bottom of Funnel)에 집중하면,
lead·체류 등 신뢰 시그널이 더 강해진다.
어떻게
  1. 자체 홈페이지 SEO 방치 → Ghost CMS 구축 + SEO 기본 재정비
  2. 발행 전략 분리 — 일반: AI 주 2회 (검색 커버리지) / 코어: 직접 주 1회 (의사결정자 타겟)
  3. 코어 콘텐츠 7편 — 신뢰·브랜딩 콘텐츠 (제품 소개·케이스 스터디·CEO voice). 광고 랜딩 페이지로 활용
    AX, AI 암묵지, Luffy, 인재 리스트, C-레벨, RPO, 다이렉트소싱
결과
  • 트래픽 2.2배, 체류 1.4배 (코어 119뷰·35초 vs 비코어 55뷰·25초)
  • "AX 단일 글" 평균 체류 67초 (전체 상위 3%)
  • 76편 중 인바운드 lead 발생 X건 모두 코어
    • "Luffy 소개" 단일 글 lead 2건 (becuai·GS 52g)
    • LinkedIn → 코어 경로 2건 (더모임·몬드리안AI)
    • 주말 추가 2건
  • 일반 72편 lead 0건 → 가설 "코어가 신뢰 시그널 만든다" 검증
  • 이 시그널이 축 3 가설(LP가 게이트)의 출발점
03 · Supply

리드 확보 — 광고부터 LP 개편까지

병목
광고·콘텐츠로 6개월간 약 1만+ 세션. 콜드 트래픽 전환 lead 0건, warm lead(이벤트·추천·지인)도 한 자릿수.
가설
홈 화면에 간이 문의 폼을 삽입하면, 콜드 트래픽도 lead로 전환된다.
어떻게
  1. 광고 4채널 (LinkedIn·Meta·Google·영상) 흩뿌린 상태에서 두 시트 동시 운영
    • 광고 추적 시트: 채널별 비용·CPA 모니터링
    • 채널별 유입 시트: 광고 + organic + direct 통합 문의 attribution
    • → 두 시트 교차 검증으로 "광고·콘텐츠 채널 콜드 lead 0" 확인
  2. 측정 인프라로 퍼널 진단
    • 채널별 유입 시트로 콜드/warm lead 분리 검증
    • 콜드 트래픽 전환 막힘 확인 → 홈 화면 실험 가설로 전환
  3. 1차 홈 화면 실험 (CRO 스프린트, D+0 → D+11)
    lead 발생이 시급해 개선 가능한 지점 한 번에 적용
    • 홈 화면 간이 문의 폼 신설
    • Hero 메시지 재정의
    • CTA 위치·문구 변경
    • 인입 동선 재설계
  4. 2차 홈 개편 — 시각·리드 마그넷 보강
    • Hero: 이미지 → 리모션 영상
    • 리드 마그넷 신설 (AI 제작 가이드북 7종)
결과
콜드 트래픽 전환 입증
  • 1차 개편 D+11 home_generate_lead 5건
  • 2차 개편 후 D+15까지 9건 누적
  • 가설 검증: 홈 화면 폼이 콜드 lead 전환 작동
운영체제 → 코어 콘텐츠 → 홈 메인 개편.
세 축이 사이클로 작동해 6개월 콜드 lead 0 → 15일만에 9건.
